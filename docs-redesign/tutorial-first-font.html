Your first variable font

Make a real, working variable font — two letters, a Weight axis, and a slider that morphs them from Regular to Black — and install it on your computer. No drawing skills needed.

⏱ ~30 minutes 🎚 Beginner macOS · Windows · Linux
What you'll end up with

A .ttf variable font with a working Weight axis and two named styles, installed and usable in any app on your computer.

1. Install & create a font

▶
🎬 download → New font → browser opens
  1. Download Fontra Pak
    Grab it from fontra.xyz and install it.
  2. Launch it and click New font
    Name it Hello, pick the .fontra format, choose where to save.
  3. Your browser opens
    This is Fontra — running in your browser, but everything stays on your computer. No internet needed, nothing uploaded.
✓ Checkpoint

A browser tab named after your font, showing an empty glyph overview.

2. Create your first glyph

We'll start with H — the friendliest letter in type design.

  1. Open the editor view and the Preview text panel; type H.
  2. The canvas shows a grey, empty placeholder. Double-click it and confirm Create. Fontra assigns the right Unicode for you.

3. Draw H and o — no Bézier required

▶
🎬 three rectangles → H · two ellipses → o

The H — three rectangles

Pick the Shape tool (rectangle), and use the Selection info panel to type exact values:

Partxywidthheight
Left stem600110700
Right stem4300110700
Crossbar60300480100

Select all (⌘A) and apply Remove overlap so the three boxes fuse into one H. Set the advance width to 600.

The o — two ellipses

Add o in Preview text and create it. With the Shape tool in ellipse mode, draw an outer oval (x 40, y −10, w 420, h 520 — the overshoot below the baseline is intentional) and an inner oval (x 140, y 90, w 220, h 320). Select both and Subtract to punch the counter. Advance width 500.

✓ Checkpoint

Typing HoHoHo in Preview text shows your two letters spelling like a font.

4. Add a Weight axis

▶
🎬 Axes dialog: New axis

Open Font → Axes → New axis… and fill in:

Name / UI NameWeight
OT tagwght
Min / Default / Max400 / 400 / 900

Under Axis values, add two labels: Regular = 400 and Black = 900. The export turns these into the font's named instances automatically.

Why 400–900, not 100–900?

An axis range should only span where you have actual designs. You'll draw a Black today; you won't draw a Thin. Extend the range the day you do.

5. Draw the Black

▶
🎬 virtual source → fatten stems
  1. Font → Sources → New source…: name Black, location Weight = 900.
  2. In the Designspace panel, the Black source shows in grey — a virtual source. Double-click it to copy your current drawing there.
  3. With the slider at 900, drag the inner stem edges outward to fatten the H. Repeat for the o.
⚠ One rule makes interpolation work

Every source must have the same contours, same order, same number of points. That's why we copied the Regular and pushed its points around. If sources diverge, Fontra shows a bug icon next to the source.

6. The magic

▶
🎬 the slider gliding, letters breathing

Type HoHoHo and drag the Weight slider slowly from 400 to 900. Your letters glide through every weight in between. Stop at 547 — that weight exists now, and nobody drew it. This is what variable-first means: you never configured interpolation, you never left it.

7. Export and install

▶
🎬 Export as → TTF → installed
  1. File → Export as… → TrueType (.ttf) → save.
  2. Install it (double-click → Install Font).
  3. Open any app with a variable-font slider, or drop the file on wakamaifondue.com. Type HoHoHo. Try every weight.
🎉 You made a variable font in half an hour.

Real, installable, with a working axis and two named styles.
