Fontra Workflow

A command-line tool for processing fonts in repeatable, declarative pipelines. Describe what should happen in a YAML file — read, filter, merge, write — and run it.

New section

Workflow is powerful but currently undocumented on the live site. This is a proposed home for it: a recipe book up front, a generated action reference behind. Why →

A first pipeline

# build.yaml
steps:
  - input: fontra-read
    source: MyFont.designspace
  - filter: subset-glyphs
    glyphNamesFile: latin-basic.txt
  - output: compile-fontmake
    destination: MyFont-Subset.ttf
$ fontra-workflow build.yaml --output-dir build

How it works

  • Five step types — input, filter, output, fork, fork-merge.
  • Lazy & pull-based — outputs pull glyphs through the chain; only what's written gets processed.
  • Composable — every filter is itself a font, so any output can feed any input.
  • Extensible — third-party actions register through an entry point (this is how fontra-compile adds its compilers).
